Services

service Keys

Provides methods for working with pre-keys.

rpc GetPreKeyCount
Request: org.signal.chat.keys.GetPreKeyCountRequest
Response: org.signal.chat.keys.GetPreKeyCountResponse

Retrieves an approximate count of the number of the various kinds of pre-keys stored for the authenticated device.

rpc GetPreKeys
Request: org.signal.chat.keys.GetPreKeysRequest
Response: org.signal.chat.keys.GetPreKeysResponse

Retrieves a set of pre-keys for establishing a session with the targeted device or devices. Note that callers with an unidentified access key for the targeted account should use the version of this method in KeysAnonymous instead.

rpc SetOneTimeEcPreKeys
Request: org.signal.chat.keys.SetOneTimeEcPreKeysRequest
Response: org.signal.chat.keys.SetPreKeyResponse

Uploads a new set of one-time EC pre-keys for the authenticated device, clearing any previously-stored pre-keys. Note that all keys submitted via a single call to this method must have the same identity type (i.e. if the first key has an ACI identity type, then all other keys in the same stream must also have an ACI identity type). The provided list of pre-keys must be non-empty.

rpc SetOneTimeKemSignedPreKeys
Request: org.signal.chat.keys.SetOneTimeKemSignedPreKeysRequest
Response: org.signal.chat.keys.SetPreKeyResponse

Uploads a new set of one-time KEM pre-keys for the authenticated device, clearing any previously-stored pre-keys. Note that all keys submitted via a single call to this method must have the same identity type (i.e. if the first key has an ACI identity type, then all other keys in the same stream must also have an ACI identity type). The provided list of pre-keys must be non-empty.

rpc SetEcSignedPreKey
Request: org.signal.chat.keys.SetEcSignedPreKeyRequest
Response: org.signal.chat.keys.SetPreKeyResponse

Sets the signed EC pre-key for one identity (i.e. ACI or PNI) associated with the authenticated device.

rpc SetKemLastResortPreKey
Request: org.signal.chat.keys.SetKemLastResortPreKeyRequest
Response: org.signal.chat.keys.SetPreKeyResponse

Sets the last-resort KEM pre-key for one identity (i.e. ACI or PNI) associated with the authenticated device.

service KeysAnonymous

Provides methods for working with pre-keys using "unidentified access" credentials.

rpc GetPreKeys
Request: org.signal.chat.keys.GetPreKeysAnonymousRequest
Response: org.signal.chat.keys.GetPreKeysAnonymousResponse

Retrieves a set of pre-keys for establishing a session with the targeted device or devices. Callers must not submit any self-identifying credentials when calling this method and must instead present the targeted account's unidentified access key as an anonymous authentication mechanism. Callers without an unidentified access key should use the equivalent, authenticated method in Keys instead.

rpc CheckIdentityKeys
Request: org.signal.chat.keys.CheckIdentityKeyRequest
Response: org.signal.chat.keys.CheckIdentityKeyResponse

Checks identity key fingerprints of the target accounts.

Returns a stream of elements, each one representing an account that had a mismatched identity key fingerprint with the server and the corresponding identity key stored by the server.
