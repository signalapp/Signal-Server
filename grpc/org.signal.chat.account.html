Services

service Accounts

Provides methods for working with Signal accounts.

rpc GetAccountIdentity
Request: org.signal.chat.account.GetAccountIdentityRequest
Response: org.signal.chat.account.GetAccountIdentityResponse

Returns basic identifiers for the authenticated account.

rpc DeleteAccount
Request: org.signal.chat.account.DeleteAccountRequest
Response: org.signal.chat.account.DeleteAccountResponse

Deletes the authenticated account, purging all associated data in the process.

rpc SetRegistrationLock
Request: org.signal.chat.account.SetRegistrationLockRequest
Response: org.signal.chat.account.SetRegistrationLockResponse

Sets the registration lock secret for the authenticated account. To remove a registration lock, please use ClearRegistrationLock.

rpc ClearRegistrationLock
Request: org.signal.chat.account.ClearRegistrationLockRequest
Response: org.signal.chat.account.ClearRegistrationLockResponse

Removes any registration lock credentials from the authenticated account.

rpc ReserveUsernameHash
Request: org.signal.chat.account.ReserveUsernameHashRequest
Response: org.signal.chat.account.ReserveUsernameHashResponse

Attempts to reserve one of multiple given username hashes. Reserved usernames may be claimed later via ConfirmUsernameHash.

rpc ConfirmUsernameHash
Request: org.signal.chat.account.ConfirmUsernameHashRequest
Response: org.signal.chat.account.ConfirmUsernameHashResponse

Sets the username hash/encrypted username to a previously-reserved value (see ReserveUsernameHash).

rpc DeleteUsernameHash
Request: org.signal.chat.account.DeleteUsernameHashRequest
Response: org.signal.chat.account.DeleteUsernameHashResponse

Clears the current username hash, ciphertext, and link for the authenticated user.

rpc SetUsernameLink
Request: org.signal.chat.account.SetUsernameLinkRequest
Response: org.signal.chat.account.SetUsernameLinkResponse

Associates the given username ciphertext with the account, replacing any previously stored ciphertext. A new link handle will optionally be created, and the link handle to use will be returned in any event.

rpc DeleteUsernameLink
Request: org.signal.chat.account.DeleteUsernameLinkRequest
Response: org.signal.chat.account.DeleteUsernameLinkResponse

Clears any username link associated with the authenticated account.

rpc ConfigureUnidentifiedAccess
Request: org.signal.chat.account.ConfigureUnidentifiedAccessRequest
Response: org.signal.chat.account.ConfigureUnidentifiedAccessResponse

Configures "unidentified access" keys and preferences for the authenticated account. Other users permitted to interact with this account anonymously may take actions like fetching pre-keys and profiles for this account or sending sealed-sender messages without providing identifying credentials.

rpc SetDiscoverableByPhoneNumber
Request: org.signal.chat.account.SetDiscoverableByPhoneNumberRequest
Response: org.signal.chat.account.SetDiscoverableByPhoneNumberResponse

Sets whether the authenticated account may be discovered by phone number via the Contact Discovery Service (CDS).

rpc SetRegistrationRecoveryPassword
Request: org.signal.chat.account.SetRegistrationRecoveryPasswordRequest
Response: org.signal.chat.account.SetRegistrationRecoveryPasswordResponse

Sets the registration recovery password for the authenticated account.

rpc SetZkCredentialKey
Request: org.signal.chat.account.SetZkCredentialKeyRequest
Response: org.signal.chat.account.SetZkCredentialKeyResponse

Store a public key used to issue and verify zero-knowledge (anonymous) credentials for the account.

service AccountsAnonymous

Provides methods for looking up Signal accounts. Callers must not provide identifying credentials when calling methods in this service.

rpc CheckAccountExistence
Request: org.signal.chat.account.CheckAccountExistenceRequest
Response: org.signal.chat.account.CheckAccountExistenceResponse

Checks whether an account with the given service identifier exists.

rpc LookupUsernameHash
Request: org.signal.chat.account.LookupUsernameHashRequest
Response: org.signal.chat.account.LookupUsernameHashResponse

Finds the service identifier of the account associated with the given username hash.

rpc LookupUsernameLink
Request: org.signal.chat.account.LookupUsernameLinkRequest
Response: org.signal.chat.account.LookupUsernameLinkResponse

Finds the encrypted username identified by a given username link handle.

Field Type Description
1 unidentified_access_key bytes

The key that other users must provide to interact with this account anonymously (i.e. to retrieve keys or profiles or to send messages) unless unrestricted unidentified access is permitted. Must be present if unrestricted unidentified access is not allowed.

message ConfirmUsernameHashRequest

Field Type Description
1 username_hash bytes

The username hash to claim for the authenticated account.
2 zk_proof bytes

A zero-knowledge proof that the given username hash was generated by the Signal username algorithm.

The ciphertext of the chosen username for use in public-facing contexts (e.g. links and QR codes).

3 username_not_available org.signal.chat.account.UsernameNotAvailable

message SetZkCredentialKeyResponse

Field Type Description

A random, non-zero, value that must be included in credential requests using the key.

This value allows the server to ratchet the resulting binding identity, as reverting to the previous key will result in a new rotation ID.
